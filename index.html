デプロイメントとは

📝 定義

デプロイメント（deployment）は、ビルドされたアプリケーションを本番環境に配置するプロセスです。

「自分のパソコンで動いているアプリを、サーバーやクラウド環境に置く」という理解で大丈夫です。

🔄 プロセス全体

  1. 1. コードの準備 - テストが通っているか確認
  2. 2. ビルド - ソースコードを実行可能な形に変換
  3. 3. 環境設定 - 環境変数やDB接続を設定
  4. 4. 配置 - 本番環境にアップロード（これがデプロイ）

Webサーバーの基本

リクエスト ↔ レスポンス

1️⃣ ユーザーがブラウザでURLを入力

例：https://example.com

↓

2️⃣ ブラウザが「リクエスト」を送信

「HTMLファイルをください」

↓

3️⃣ サーバーが「レスポンス」を返信

HTMLファイル、CSS、JavaScriptなどを送信

↓

4️⃣ ブラウザが表示

受け取ったファイルを解析してWebページを表示

📦 Webサーバーの役割

  • ✓ HTTPリクエストを受け取る
  • ✓ ファイルを検索する
  • ✓ ファイルをブラウザに送信
  • ✓ エラー処理（404など）
  • ✓ アクセス制御

🔧 Webサーバーの例

  • Apache - 最も使われている
  • nginx - 軽量で高速
  • IIS - Windows向け
  • Caddy - モダンで簡単

従来のデプロイ vs 現代のデプロイ

❌ 従来のデプロイ

FTP + レンタルサーバー

FileZillaなどでファイルを手動アップロード

  • 🚫 手動でのアップロード（ミスが多い）
  • 🚫 セキュリティが弱い（FTPは暗号化なし）
  • 🚫 バージョン管理がない
  • 🚫 自動化できない
  • 🚫 スケーリングが難しい

✅ 現代のデプロイ

Git連携 + クラウドホスティング

GitHubにpushするだけで自動デプロイ

  • ✅ 自動デプロイ（人為的ミスなし）
  • ✅ セキュリティが高い（HTTPS自動）
  • ✅ バージョン管理完全（Git）
  • ✅ CI/CD完全自動化
  • ✅ 世界規模のCDNで高速配信

まとめ

🎯 現代のデプロイの特徴

  • ✅ Git連携：コード変更が自動的に本番環境に反映
  • ✅ 完全な自動管理：インフラの設定や運用は不要
  • ✅ 世界規模のCDN：どの地域からでも高速アクセス
  • ✅ 無料でスタート：学習段階でコストなし

プログラミング初心者にとって、現代的なデプロイツールの登場は非常に重要です。

かつては、素晴らしいアプリケーションを作ってもインフラ知識がなければ公開できませんでした。

今は、コードを書く能力さえあれば、インフラの心配なく世界中に向けてアプリケーションを公開できます！
